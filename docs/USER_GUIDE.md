# Руководство пользователя — СМР ПСД

> Для инженеров ПТО и СДО. Написано простым языком.

---

## Общие сведения

**СМР ПСД** — веб-приложение для управления проектно-сметной документацией (ПСД), исполнительной документацией (ИД), договорами и отчётностью на объектах строительства.

### Кто пользуется системой

| Роль                  | Задачи                                                       |
|-----------------------|--------------------------------------------------------------|
| **Инженер ПТО**       | Сметы, договоры, факт выполнения, АОСР, М-29, отчёты        |
| **Инженер СДО**       | Предписания, контроль ИД, дашборд готовности                 |
| **Руководитель**      | Управленческие отчёты, дашборды                              |

### Как войти

Откройте в браузере адрес, указанный администратором (обычно `http://<адрес_сервера>:3000`). Вы попадёте на страницу **Проекты**.

---

## Инженер ПТО — порядок работы

### 1. Создать проект

**Страница:** Проекты → кнопка **«Новый проект»**

Заполните:
- **Шифр** — уникальный код проекта (например, `2024-КЖ-001`)
- **Наименование** — полное название объекта
- **Заказчик** — организация-заказчик
- **Подрядчик** — организация-подрядчик
- **Статус** — `Проектирование`, `Строительство`, `Ввод в эксплуатацию` или `Завершён`

Нажмите **«Сохранить»**. Проект появится в списке. Кликните по названию — откроется рабочая область проекта с боковым меню.

---

### 2. Создать сметную иерархию

**Страница:** боковое меню → **Сметы**

Сметная структура имеет 5 уровней:

```
ССР (Сводный сметный расчёт)
 └── ОСР (Объектная сметная расчёт)
      └── ЛСР (Локальный сметный расчёт)
           └── ЛС (Локальная смета — основная или дополнительная)
                └── Сметная статья (позиция: шифр, наименование, единица, объём, сумма)
```

**Как создать:**
1. В дереве смет нажмите **«+»** на нужном уровне
2. Заполните шифр и наименование
3. Для ЛСР укажите **индекс пересчёта цен**
4. Для ЛС выберите тип: **Основная** или **Дополнительная**
5. Добавьте сметные статьи в ЛС — это конечные позиции с объёмами и суммами

**Цветовая индикация остатков:**
- 🟢 Зелёный — остаток ≥ 10% лимита
- 🟡 Жёлтый — остаток < 10% лимита
- 🔴 Красный — перерасход (остаток < 0)

---

### 3. Добавить чертежи

**Страница:** боковое меню → **Чертежи**

1. Нажмите **«Добавить»**
2. Заполните:
   - **Марка** — марка чертежа (КЖ, АР, ЭОМ, ОВ, ВК и т.д.)
   - **Наименование** — название листа
   - **Номер листа** и **Ревизия** (необязательно)
   - **Статус:** `Черновик` → `Выдан в производство` → `Заменён`
3. Загрузите файл (PDF/DWG) — файл сохраняется в хранилище

> **Важно:** Установите статус **«Выдан в производство работ»** для чертежей, по которым ведутся работы.

---

### 4. Распределить ЛС по договорам

**Страница:** боковое меню → **Договоры** → вкладка **«Распределение ЛС → Договоры»**

Для каждой ЛС укажите, какой процент объёма покрывается каким доходным договором.

**Правила:**
- Суммарный процент по одной ЛС **не может превышать 100%**
- Если SUM > 100%, кнопка **«Сохранить»** заблокирована
- Незакрытый остаток отображается в рублях

После привязки ЛС → доходный договор, привяжите доходные договоры к расходным на соответствующих вкладках.

---

### 5. Ежемесячный ввод факта выполнения

**Страница:** боковое меню → **Сметы** → блок **«Быстрый ввод факта»** (справа)

Это самая частая операция ПТО. Форма позволяет ввести факт **без навигации по дереву смет**:

1. **Выберите ЛС** из выпадающего списка
2. **Выберите сметную статью** — список загружается автоматически
3. **Выберите период** — месяц и год
4. Введите **объём (факт)** и **сумму (факт)** в рублях
5. При необходимости укажите **номер КС-2**
6. Нажмите **«Сохранить факт»**

После сохранения дерево смет и панель уведомлений обновятся автоматически.

---

### 6. Оформить АОСР / АООК

**Страница:** боковое меню → **Исп. документация**

1. Нажмите **«Создать акт»** (кнопка в правом верхнем углу)
2. Заполните:
   - **Номер** и **Наименование** акта
   - **Тип:** АОСР (скрытые работы) или АООК (ответственные конструкции)
   - **Дата работ** и **Дата подписания**
   - **Прораб** и **Технадзор** (ФИО)
3. В мультиселекте выберите **чертежи** и **спецификации**, к которым относится акт
4. Нажмите **«Сохранить»**
5. Загрузите скан подписанного акта

**Фильтры:** используйте фильтры по типу, статусу, дате и марке чертежа для поиска актов.

---

### 7. Вести М-29 (расход материалов)

1. Создайте отчёт М-29 за месяц: укажите проект, период, ФИО прораба
2. Заполните строки отчёта:
   - Позиция спецификации
   - Нормативный расход
   - Фактический расход
   - Отклонение рассчитывается автоматически
3. При отклонении > 0 — перерасход помечается красным

---

### 8. Генерировать отчёты

**Страница:** боковое меню → **Отчёты**

1. Выберите **тип отчёта** из выпадающего списка (8 групп, 21 тип)
2. Заполните **параметры** — поля появляются динамически в зависимости от типа
3. Нажмите **«Сформировать отчёт»**
4. Дождитесь генерации (индикатор прогресса)
5. Файл **скачается автоматически** (Excel или PDF)

**Ранее сформированные отчёты** отображаются в таблице ниже — их можно скачать повторно или удалить.

### Доступные типы отчётов

| Группа                    | Отчёты                                                              |
|---------------------------|---------------------------------------------------------------------|
| Финансовые                | Остатки лимита, Лимиты по договорам, Помесячное выполнение, Поквартальное |
| Приёмочные формы          | КС-2, КС-3, КС-6а                                                  |
| Материальная отчётность   | М-29, Потребность в материалах                                      |
| Статусы документации      | Реестр ИД, Готовность ИД, Предписания, Сертификаты                  |
| Сроки и планирование      | Отклонение от графика                                               |
| Журналы СДО               | Общий журнал работ, Журнал входного контроля                        |
| Договорные                | Реестр договоров, Маржа, Незакрытые ЛС                              |
| Управленческие            | Управленческий отчёт (1 стр. А4), Сводная по всем проектам          |

## Инженер СДО — порядок работы

### 1. Создать предписание

**Страница:** боковое меню → **Предписания**

1. Создайте предписание:
   - **Номер** — номер предписания
   - **Описание нарушения** — подробное описание
   - **Ссылка на норматив** — СП, ГОСТ, СНиП и т.д.
   - **Плановый срок устранения** — дата
2. Приложите фото нарушения (если нужно)
3. Статус: `Открыто`

---

### 2. Контролировать устранение

**Страница:** боковое меню → **Предписания**

По мере устранения нарушений:

1. Нажмите **«Статус»** у нужного предписания
2. Выберите новый статус:
   - `Открыто` → `В работе` → `Устранено` → `Закрыто`
3. Добавьте **комментарий** (обязателен при смене статуса)
4. Нажмите **«Применить»**

**История изменений:** нажмите **«История»** — откроется боковая панель с таймлайном всех изменений статуса, авторов и комментариев.

**Просроченные предписания** выделяются красным фоном и иконкой ⚠️.

---

### 3. Контролировать уведомления

**Панель уведомлений** отображается справа на странице Предписания:

| Иконка | Описание                                  |
|--------|-------------------------------------------|
| 🔴     | Просроченные предписания (срок < сегодня) |
| 🟡     | Сертификаты с истекающим сроком (< 30 дней)|
| 🟠     | ЛС с нераспределёнными объёмами (< 100%)  |
| 🔴     | Перерасход бюджета (остаток < 0)          |

---

### 4. Контролировать готовность ИД

**Страница:** боковое меню → **Исп. документация** → блок **«Готовность ИД»** (справа)

Дашборд показывает прогресс-бары по каждому разделу (марке чертежей):
- Сколько АОСР требуется (по количеству чертежей)
- Сколько АОСР оформлено
- Процент готовности

Сводный процент готовности проекта отображается в верхней части дашборда.

---

### 5. Сформировать отчёт по предписаниям

**Страница:** боковое меню → **Отчёты**

1. Выберите тип: **«Сводный отчёт по предписаниям СДО»**
2. Укажите параметры: проект, статус (необязательно), период (дата с / дата по)
3. Нажмите **«Сформировать»**
4. Скачайте Excel-файл с итогами по статусам и списком просроченных

---

## Частые вопросы

### Почему кнопка «Сохранить» заблокирована при распределении ЛС?

Суммарный процент распределения превышает 100%. Уменьшите долю одного из договоров.

### Почему остаток лимита красный?

Фактическая сумма выполнения превысила утверждённый лимит. Проверьте введённые данные или согласуйте увеличение лимита.

### Как обновить статус чертежа?

Страница Чертежи → кнопка редактирования (✏️) → измените статус → Сохранить.

### Как удалить ранее сформированный отчёт?

Страница Отчёты → таблица «Ранее сформированные» → кнопка **«Удалить»** у нужного отчёта.
